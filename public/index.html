1

Data Ingestion

Kafka producer fetches BTC/USDT OHLCV candles from Binance via CCXT with deduplication and delivery guarantees.

2

Staging Load

Kafka consumer batch-inserts into PostgreSQL staging with at-least-once semantics and dead letter queue for failures.

3

OHLCV Aggregation

PySpark job aggregates minute-level data into hourly and daily candles with proper first/last open/close and VWAP.

4

Technical Indicators

PySpark computes SMA, EMA, RSI, MACD, Bollinger Bands, ATR, and OBV using window functions.

5

dbt Modeling

Three-layer dbt project: staging (validation), intermediate (aggregation), marts (analytics-ready facts and dimensions).

6

Orchestration

Airflow DAG schedules the full pipeline daily with task groups, retries, metadata tracking, and health checks.

Key Features.

📨

Kafka Streaming

Confluent Kafka with Snappy compression, delivery guarantees, consumer groups, manual offset commits, and a dead letter queue.

⚡

PySpark Processing

Distributed batch processing with window functions for time-series aggregation. Reads/writes PostgreSQL via JDBC.

📊

dbt Analytics

Staging, intermediate, and mart layers with custom macros, schema tests, and data quality assertions.

🕒

Airflow Orchestration

DAG with task groups, CeleryExecutor for parallel tasks, retries, execution timeouts, and pipeline metadata tracking.

🛡

Data Quality

Input validation, deduplication (upserts), dead letter queue for parse failures, dbt tests, and health check endpoints.

🐳

Docker Compose

10-service stack: Kafka, Zookeeper, PostgreSQL, Redis, Spark master/worker, Airflow webserver/scheduler/worker, init containers.
